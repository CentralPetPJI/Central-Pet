The current storage code is the main anti-pattern to replace:

```ts
notes: `Tutor: ${formData.tutor}. Cidade: ${formData.city}. Contato: ${formData.contact}.`,
```

The frontend pet model should move to explicit nested fields:

```ts
responsibleProfile: ResponsibleProfile;
structuredContact: StructuredContact;
trustSignals: TrustSignals;
```

Keep current UI-facing fields like `name`, `species`, `photo`, and `behavioralCharacteristics`, but stop using `notes` as pseudo-schema.
</interfaces>
