<interfaces>
The current backend pet contract is flat and ownership-heavy:

```ts
responsibleUserId: string;
city?: string;
state?: string;
```

This plan should add explicit nested structures while preserving `responsibleUserId` only as an ownership pointer:

```ts
responsibleProfile: ResponsibleProfile;
structuredContact: StructuredContact;
trustSignals: TrustSignals;
```

Adoption contact should be scaffolded with:

```ts
petId: string;
adopterName: string;
adopterContact: string;
message?: string;
preferredChannel: ContactChannel;
contextSnapshot: CanonicalPetContactSnapshot;
```

</interfaces>
